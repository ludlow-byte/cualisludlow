Critique cuatower.com Using Nielsen’s Heuristics

  • Visibility of system status – the system shows the processing circle of dots on the internet tab; updates and outputs are delivered quickly.
  • User control and freedom – the “Subscribe” and “Advertise” options in the subject directory open new tabs. It is not immediately clear that these are new tabs and that the user needs to switch back to the original site. A pop-up warning might be a nice feature to help users figure out how get back to their searches.
  • Error prevention – there is a simple spell check function operating with the search input box.
  • Recognition rather than recall – the website heading, subject directory, search icon and box, and 10-year archives are available from every page.
  • Aesthetics and minimalist design – the subject pages, navigated to via the directory, don’t all match the home page’s format. The Politics page comes the closest to matching the home page. The “Flash Story” section on the home page seems redundant to the “What’s New” and “Featured Story” sections and the scrolling titles are visually distracting.
  • Help users recognize, diagnose, and recover from errors – although there is a spell-check function within the search box, the system does not provide suggestions for alternative search terms or spellings (see Figure 2).
  • Help and documentation – there is no “help” option or FAQ page on website.

Strengths and Weaknesses

As we’re starting to see, The Tower website has a mixed bag of characteristics. The website does a few things well withing Nielsen’s 4th heuristic, Consistency and Standards, and 2nd heuristic, Match Between Systems and the Real World.

  • Consistency and standards – the top of the home page is consistent with industry standards, with a subject-based directory that includes subscription and donation options immediately below the website heading and a search icon to the far right of the directory line.
  • Match between system and the real world – the menu labels are straightforward with simple drop-down options, and the website does not appear to use jargon from the journalism world or the information, communication and technology world. The system appears to list search results by some version of “most relevant” first and then from newest to oldest.

However, as good as the website does within the 2nd heuristic, it also has some not-insignificant weaknesses in the same category. It could also use some improvements in the 7th heuristic, Flexibility and Efficiency of Use.

  • Match between system and the real world – the Archive section of cuatower.com only goes back to September 2015 and it does not provide a link to the older archives stored on JSTOR.
    • The Tower mimics a negative aspect of real-world journalism - The Tower appears to have published “native advertising” at least once, “Best Car Air Purifier, EraClean Mini Air Purifier For Your Home and Car” (The Tower, February 9, 2020). The domain that was the likely source of the article is no longer operational and is for sale (see Figure 3).
  • Flexibility and efficiency of use – article selection is generally flexible with access provided on the home page by direct link, by navigating the directories, or by search box. However, efficiency seems to be less of a concern. Users cannot narrow down initial search results and must review each retrieved item for relevance. There is no option to create or save frequently used search terms. Lack of a filter option and the ability to save are two strong strikes against the website from the perspective of a teacher or student that needs to conduct research using The Tower.

    Part 2 - Comparing Student Heuristic Analysis of cuatower.com With Mini-Usability Test of The Tower Archives

    The primary comparison to be made between the current site and the archive site is about the search and filter functions available on each site. The archive site can be searched using a keyword or phrase, a date range, or by language (although The Tower is only available in English). The mini-usability test revealed that using a keyword did not always retrieve the best, or any, results. However, searching the archives by date was a much more effective option for the user. This remains a good option only so long as the user knows the relevant date or date range of their research topic. The current website has the opposite problem: a keyword search will provide the broadest range of related results, while using the current site’s archive section forces the user to search each issue for a relevant article.

    The first step to improving the retrievability of articles from The Tower is to provide a link on both the current website and the JSTOR archive website to the other site. The second step towards improved retrievability is adding filtering options to the current site’s search results. The current site has ten and a half years of articles available. It would be helpful to be able to search using both a keyword and a date range to limit the number of results through which a prospective teacher or student researcher needs to wade. Finally, it may be time for CUA to consider a new archive vendor given JSTOR’s age and the glitches experienced by the mini-usability tester.
